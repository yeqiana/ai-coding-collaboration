# AI Coding Guardrails

> 让 Codex / Claude Code / Cursor 写代码时，不乱改、不漏测、不写废话。

这是一个面向日常开发的 AI Coding Skill。

它不是重型 PRD 流程，也不是完整项目管理系统。  
它的目标很简单：

- 小任务不要走完整 SOP
- 大任务不要跳过分析
- AI 不要顺手重构
- 写完代码必须做漏项自检
- 没验证不能说可合并
- 最后用一屏 Executive Summary 交付

## 为什么需要它

AI 写代码很快，但经常出现这些问题：

- 一个小按钮也生成一堆文档
- 顺手改了无关文件
- 接口写了，前端没接
- DTO 改了，调用方没改
- 页面写了，路由没加
- 没跑测试，却说完成
- Review 输出几十条，看着很累

这个 Skill 用一套轻量规则解决这些问题。

## 核心机制

### 1. 任务分级

| 等级 | 适用场景 | 流程 |
|---|---|---|
| L0 微任务 | 改文案、改颜色、小 Bug | 直接改 + 自测 + 摘要 |
| L1 小功能 | 按钮、简单接口、表单校验 | Task Card + 影响分析 + Checklist |
| L2 模块任务 | 登录、权限、上传、多层联动 | 简版方案 + 任务拆分 + Review + 测试 |
| L3 项目级任务 | 新项目、大重构、架构变更 | PRD + 架构 + Backlog + Release |

关键原则：

> 不要把 L0 / L1 当 L3 做。

### 2. 最小 Diff 原则

AI 必须：

- 改最少文件
- 改最少代码
- 不做无关格式化
- 不顺手优化
- 不顺手重构
- 不随意改接口、DTO、路由、数据库字段、配置项

### 3. 影响分析

L1 以上任务，实施前必须说明是否涉及：

```text
后端接口
Service 逻辑
DTO / 类型
数据库 / 配置
前端页面 / API 调用
路由 / 导航
测试 / 验证
```

### 4. 测试分级

不用输出几十条测试点，只按 P0 / P1 / P2 分级。

| 等级 | 含义 |
|---|---|
| P0 | 必须验收：核心流程、数据正确、接口/页面入口、构建/类型检查 |
| P1 | 建议验收：空数据、异常参数、重复提交、网络异常 |
| P2 | 可选验收：性能、大数据量、极端 UI 边界 |

### 5. 漏项自检

写完代码后，AI 必须检查：

```text
1. 是否有需求未实现？
2. 是否有代码改了但没验证？
3. 是否有接口、路由、DTO、配置、数据库遗漏？
4. 是否有测试未执行？
5. 当前是否可以进入 Review？
```

### 6. Executive Summary

每轮结束必须输出：

```text
Goal:
Changed:
Validated:
Risks:
Conclusion:
```

如果没有验证核心路径，不能写“merge-ready / 可合并”。

## 快速使用

把 `SKILL.md` 放到你的 AI 工具 skills 目录，例如：

```text
~/.claude/skills/ai-coding-guardrails/SKILL.md
```

然后这样使用：

```text
使用 AI Coding Guardrails。

任务：
给用户列表新增导出按钮。

限制：
- 不改后端接口
- 不重构组件
- 保持最小 Diff
- 做漏项自检
- 没有 P0 验证不要写可合并
```

Codex 用户也可以继续使用当前仓库 Skill 名：

```text
使用 $ai-coding-collaboration。
```

## 适合场景

- 日常功能开发
- Bug 修复
- 代码 Review
- 测试验收规划
- Codex / Claude Code 执行约束
- 生成项目级 AGENTS.md / CLAUDE.md / PROJECT_RULES.md

## 不适合场景

- 法务合规最终审查
- 完整产品 PRD
- 深度安全审计
- 复杂架构委员会评审

## 一句话

> 让 AI Coding 从“看起来完成了”变成“有证据地完成了”。
